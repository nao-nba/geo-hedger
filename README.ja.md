# Geo-Hedger (ジオ・ヘッジャー)

地政学的パラダイムシフトの時代に向けて設計された、動的な資産リバランスとシナリオプランニングのための戦略的CLIツール。

*(※ 英語版のREADMEは [こちら](./README.md) をご覧ください。)*

---

## 1. 開発背景とプロダクトの哲学 (Philosophy)

**「全世界株式に思考停止で丸投げすれば安心」という時代は終わった。**

全世界株式への投資というのは、グローバルサプライチェーンに支えられた平和な世界を前提に利益を得るという手法です。しかし、世界は今アメリカの帝国覇権の終焉を伺わせる歴史的転換点であり、ブロック化した新システムへのパラダイムシフトが進行しています。資源持たざる国に住む個人にとって、「自分の国が東西どちらのシステムに帰属するのか？」「急激なインフレの中でどのように資産を守るのか？」を問われる、綱渡りの選択を迫られています。

*Geo-Hedger* は、中央集権的な「世間の正解」に依存せず、ユーザー自身が地政学的シナリオを予測し、主体的にポートフォリオを機動リバランスするための戦略的CLI（Command Line Interface）ツールです。

*(※ 今後、Next.jsをフロントエンドに採用し、UXを強化する予定です。また、今後予想される世界的なエネルギー不安による不安定な環境を想定し、データはJSONで保持する選択をしています。これによりオフラインでの回復力と、自己主権的なデータ管理を実現します。)*

---

## 2. コア機能 (Core Features)

### ① 動的なシナリオ（フェーズ）管理
* ユーザーが想定する地政学的シナリオを「フェーズ」として定義。
* 「なぜその比率にするのか」の理由（Rationale）と、具体的な行動方針（Action）、想定されるリスク（Risk）を言語化して保存可能。

### ② 家系全体の資産の一元管理（マルチ・アカウント対応）
* 家族間や、複数の証券会社・ウォレットに分散した資産を、誰が・どこに・どれだけ持っているか紐付けて入力可能。
* これらを合算し、家系全体の真のアセットアロケーションを俯瞰。
* *目的：家族単位でこの動乱をどう生き延びるか？という最大の課題を解決するため。*

### ③ リアルタイム・レートによる資産評価（フォールバック機能付き）
* 外部（Googleスプレッドシート等）から、リアルタイムの市場レート（為替、金、株価、暗号資産）を安全に取得し、瞬時に円建て換算。
* ネットワークやエネルギーの不安定化により市場レートの取得に失敗した場合は、手動でのレート入力に対応し、ツールの継続運用を保証。

### ④ リバランス・ガイド と ネクスト・アクションの自動生成
* 「現在の資産比率」と「指定したフェーズの目標比率」の乖離を可視化。
* 単なる数値の提示にとどまらず、**「誰の・どの口座で・何を・いくら売買すべきか」**の具体的なTo-Doリスト（行動指示）まで自動で出力。

## 3. 要件定義 と データ構造

### 前提条件
* 本ツールで管理・可視化するのは、生活防衛資金を除いた「余剰資金（投資可能資金）」とする。

### データ構造 (`state.json`)
データはローカルのJSONファイルとして永続化されます。これはオフライン耐性を高め、将来的なリレーショナルデータベースへの移行を容易にする設計となっています。

```json
{
  "updated_at": "2026-04-04T10:00:00Z",
  "selected_phase_id": "phase_2",
  "phases": {
    "phase_1": {
      "name": "Phase1_平時・円安局面",
      "scenario": "現状維持。緩やかなインフレと円安が続くシナリオ。",
      "action": "現金を絞り、西側（金）と無国籍（BTC）を厚めにキープ。",
      "targets": { "JPY": 10.0, "USD": 10.0, "GLDM": 30.0, "2800": 20.0, "BTC": 30.0 }
    },
    "phase_2": {
      "name": "Phase2_有事・クラッシュ局面",
      "scenario": "地政学リスクが顕在化し、株価暴落と一時的なドル高が発生するシナリオ。",
      "action": "高値のドルを円転し、底打ちしたBTCや東側（2800）を拾う。",
      "targets": { "JPY": 30.0, "USD": 0.0, "GLDM": 20.0, "2800": 20.0, "BTC": 30.0 }
    }
  },
  "assets": {
    "家族A（SBI証券）": { "JPY": 1000000, "USD": 5000, "BTC": 0.2 },
    "家族B（楽天証券）": { "JPY": 500000, "GLDM": 50 }
  }
}
```

## 5. 使い方

### 1. 前提条件
以下の環境が必要です。
* Go (バージョン1.21以降)
* Google Spreadsheetを作成しCSV公開URLをコピー
```
列：
Symbol | Rate | Category
例：
USD | =GOOGLEFINANCE("CURRENCY:USDJPY") | currency
2800 | =GOOGLEFINANCE("HKG:2800") * GOOGLEFINANCE("CURRENCY:HKDJPY") | eastern
GLDM | =GOOGLEFINANCE("GLDM") * GOOGLEFINANCE("CURRENCY:USDJPY") | western
```
* .env.exampleを参考に、.envファイルをルートに作成。GoogleスプレッドシートのCSV公開URLを貼り付ける。
```
.env
# Google Spreadsheet CSV URL
GEO_HEDGER_CSV_URL="https://docs.google.com/spreadsheets/d/e/XXXXX/pub?output=csv"
```

### 2. インストール
```bash
git clone [https://github.com/nao-nba/geo-hedger.git](https://github.com/nao-nba/geo-hedger.git)
```
```bash
cd geo-hedger
```
```bash
go build ./cmd/geo-hedger
```

### 3. アプリ全体の基本設定（基準通貨と比較対象フェーズの指定）
```bash
./geo-hedger set-config --base-currency JPY --target-phase phase_2
```

### 4. フェーズ（シナリオと目標比率）の定義・更新
```bash
./geo-hedger set-phase --id phase_2 --name "Phase2_有事・クラッシュ" --scenario "地政学リスクの顕在化" --action "高値のドルを円転" --targets "JPY:30,USD:0,GLDM:20,BTC:30"
```

### 5. 家族の資産の追加・更新
```bash
# 追加
./geo-hedger add-asset --owner "家族A（SBI証券）" --symbol BTC --amount 0.2
```
```bash
# 一部売却
./geo-hedger add-asset --owner "家族A（SBI証券）" --symbol BTC --amount -0.2
```

### 6. 特定の家族の特定の資産を削除
```bash
# 特定の家族の特定の資産を削除する場合
./geo-hedger delete-asset --owner "家族A（SBI証券）" --symbol BTC

# 家族のデータをまるごと削除する場合
./geo-hedger delete-asset --owner "家族A（SBI証券）"
```

### 7. 状態の確認とネクスト・アクションの出力
```bash
./geo-hedger status
```
